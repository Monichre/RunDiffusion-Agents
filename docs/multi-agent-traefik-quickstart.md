# Multi-Agent Shared Host with Traefik Routing (Recommended)

Run **Hermes**, **Codex**, **Claude**, **Gemini**, and a **Quantum Filebrowser** side-by-side on a single shared host. Traefik handles hostname-based routing so each tenant gets its own isolated container, while all agents share the same ingress.

> **This is the recommended deployment model.** It gives you per-tenant isolation, centralized governance via the YAML control plane, and the ability to scale from one agent to many without re-architecting.

---

## What You Get

| Component | Route | Description |
|-----------|-------|-------------|
| **OpenClaw** | `/openclaw` | AI control plane and orchestration interface |
| **Dashboard** | `/dashboard` | Operator landing page with agent overview |
| **Hermes** | `/hermes` | Delegated async tasks — sub-agent orchestration |
| **Codex** | `/codex` | OpenAI Codex CLI for code generation |
| **Claude** | `/claude` | Anthropic Claude Code assistant |
| **Gemini** | `/gemini` | Google Gemini CLI interface |
| **Filebrowser** | `/filebrowser` | Quantum Filebrowser — secure document management |
| **Terminal** | `/terminal` | Full interactive terminal session |

All routes are protected by Basic Auth (shared credentials for `/dashboard`, `/terminal`, `/filebrowser`, and all agent routes). OpenClaw uses its own native auth or trusted-proxy mode.

## Architecture Overview

```
                    ┌──────────────────────────────────┐
                    │           Traefik                 │
   Internet ──────▶│  Host(tenant-a.example.com) ──▶ tenant-a-gateway:8080
                    │  Host(tenant-b.example.com) ──▶ tenant-b-gateway:8080
                    └──────────────────────────────────┘
                                    │
                        ┌───────────┴──────────┐
                        ▼                      ▼
               ┌─────────────────┐   ┌─────────────────┐
               │   Tenant A      │   │   Tenant B      │
               │ ┌─────────────┐ │   │ ┌─────────────┐ │
               │ │  OpenClaw   │ │   │ │  OpenClaw   │ │
               │ │  Hermes     │ │   │ │  Hermes     │ │
               │ │  Codex      │ │   │ │  Codex      │ │
               │ │  Claude     │ │   │ │  Claude     │ │
               │ │  Gemini     │ │   │ │  Gemini     │ │
               │ │  Filebrowser│ │   │ │  Filebrowser│ │
               │ │  Dashboard  │ │   │ │  Dashboard  │ │
               │ └─────────────┘ │   │ └─────────────┘ │
               └─────────────────┘   └─────────────────┘
```

Each tenant runs in its own Docker container on a shared Docker network. Traefik routes requests by `Host` header. All agents (Hermes, Codex, Claude, Gemini) run inside each tenant container as separate tmux sessions exposed via ttyd web terminals.

## Quick Start

### 1. Configure the Root Environment

```bash
cp .env.example .env
```

Edit `.env` with your host settings:

```bash
# Required settings
BASE_DOMAIN=agents.example.com      # Your domain
INGRESS_MODE=local                  # local | direct | cloudflare
DATA_ROOT=/srv/rundiffusion-agents/data
TENANT_ENV_ROOT=/srv/rundiffusion-agents/secrets/tenants

# Traefik binding
TRAEFIK_BIND_ADDRESS=127.0.0.1     # Use 0.0.0.0 for direct mode
TRAEFIK_HTTP_PORT=38080
```

### 2. Create the Tenant Registry

```bash
cp deploy/tenants/tenants.example.yml deploy/tenants/tenants.yml
```

### 3. Create Your First Tenant

```bash
./scripts/create-tenant.sh my-agent "My Agent"
```

This generates:
- A tenant registry entry in `deploy/tenants/tenants.yml`
- A tenant env file at `${TENANT_ENV_ROOT}/my-agent.env`
- A data directory at `${DATA_ROOT}/tenants/my-agent/gateway`

### 4. Configure the Tenant

Edit the generated tenant env file (`${TENANT_ENV_ROOT}/my-agent.env`):

```bash
# Identity (auto-generated)
TENANT_SLUG=my-agent
TENANT_HOSTNAME=my-agent.agents.example.com
OPENCLAW_CONTROL_UI_ALLOWED_ORIGINS=http://my-agent.agents.example.com:38080

# Auth (auto-generated — keep or rotate)
OPENCLAW_ACCESS_MODE=native
OPENCLAW_GATEWAY_TOKEN=<auto-generated>
TERMINAL_BASIC_AUTH_USERNAME=my-agent-operator
TERMINAL_BASIC_AUTH_PASSWORD=<auto-generated>

# All four agents are enabled by default
HERMES_ENABLED=1
CODEX_ENABLED=1
CLAUDE_ENABLED=1
GEMINI_ENABLED=1

# Add your API keys to activate each agent
GEMINI_API_KEY=<your-gemini-api-key>
HERMES_OPENAI_API_KEY=<your-openai-or-gemini-key>
CODEX_OPENAI_API_KEY=<your-openai-key>
CLAUDE_ANTHROPIC_API_KEY=<your-anthropic-key>
GEMINI_CLI_API_KEY=<your-gemini-cli-key>
OPENROUTER_API_KEY=<optional-openrouter-key>
```

> **Tip:** Agents without API keys still launch — they drop to an interactive shell where you can log in manually. Keys just enable non-interactive pre-auth.

### 5. Deploy

```bash
./scripts/deploy.sh
```

This:
1. Builds the shared Traefik ingress (`compose.prod.yml`)
2. Generates dynamic Traefik routing for all enabled tenants
3. Deploys each tenant container with the tenant-stack compose
4. Runs a smoke test to verify health

### 6. Verify

```bash
# Check all services
./scripts/status.sh

# Run smoke tests
./scripts/smoke-test.sh --all

# Test health endpoint directly
curl -fsS http://my-agent.agents.example.com:38080/healthz
```

## Agent Details

### Hermes — Delegated Tasks

Hermes is a Python-based AI agent that handles complex, asynchronous task delegation. Point it at a problem, and it manages sub-agent execution autonomously.

- **Route:** `/hermes`
- **Workspace:** `/data/workspaces/hermes`
- **Config:** `/data/.hermes/config.yaml`
- **Default model:** `gemini-3-flash-preview` (via Gemini OpenAI-compatible endpoint)
- **Key:** `HERMES_OPENAI_API_KEY` (accepts Gemini, OpenAI, or OpenRouter keys)

### Codex — AI Code Generation

OpenAI Codex CLI for interactive code generation and editing.

- **Route:** `/codex`
- **Workspace:** `/data/workspaces/codex`
- **Config:** `/data/.codex/config.toml`
- **Key:** `CODEX_OPENAI_API_KEY`

### Claude — AI Code Assistant

Anthropic Claude Code for pair-programming and code review.

- **Route:** `/claude`
- **Workspace:** `/data/workspaces/claude`
- **Config:** `/data/.claude/`
- **Key:** `CLAUDE_ANTHROPIC_API_KEY`

### Gemini — Google AI CLI

Google Gemini CLI for conversational AI and code tasks.

- **Route:** `/gemini`
- **Workspace:** `/data/workspaces/gemini`
- **Config:** `/data/.gemini/`
- **Key:** `GEMINI_CLI_API_KEY`

### Quantum Filebrowser

A secure file browser for managing documents and adding secrets securely.

- **Route:** `/filebrowser`
- **Scope:** `/data` directory (all workspaces and tool files)
- **Auth:** Shared Basic Auth credentials (same as `/terminal` and `/dashboard`)
- **Security:** User write permissions are OFF by default for safety. Turn them on in Filebrowser settings if you need to upload files.

> **Note:** Filebrowser runs the [Quantum Filebrowser](https://github.com/gtsteffaniak/filebrowser) fork (`ghcr.io/gtsteffaniak/filebrowser:stable-slim`), which adds performance and security improvements over the original.

## Multi-Tenant Scaling

Add more tenants at any time:

```bash
# Create a second tenant
./scripts/create-tenant.sh team-b "Team B"

# Edit the new tenant's env file
vim ${TENANT_ENV_ROOT}/team-b.env

# Deploy just the new tenant
./scripts/deploy.sh --tenant team-b

# Or deploy everything
./scripts/deploy.sh
```

Each tenant gets its own:
- Hostname (`team-b.agents.example.com`)
- Docker container with isolated `/data` volume
- Independent agent sessions (Hermes, Codex, Claude, Gemini)
- Separate auth credentials
- Filebrowser instance scoped to its own data

## YAML Control Plane (Optional)

For fleet-wide governance, create a control-plane config:

```bash
cp deploy/tenants/control-plane.example.yml ${TENANT_CONTROL_PLANE_CONFIG_PATH}
```

```yaml
tenants:
  my-agent:
    openclawVersion: 2026.3.24
    secrets:
      GEMINI_API_KEY: "<key>"
      HERMES_OPENAI_API_KEY: "<key>"
      CODEX_OPENAI_API_KEY: "<key>"
      CLAUDE_ANTHROPIC_API_KEY: "<key>"
    models:
      allowed:
        - openai-codex/gpt-5.4
      primary: openai-codex/gpt-5.4
    routes:
      gemini:
        enabled: true
```

The control plane overrides tenant env files for managed fields. Redeploy to apply:

```bash
./scripts/deploy.sh --tenant my-agent
```

## Ingress Modes

| Mode | `INGRESS_MODE` | When to Use |
|------|----------------|-------------|
| **Local** | `local` | Development, localhost, LAN access |
| **Direct** | `direct` | Public host with your own DNS + HTTPS |
| **Cloudflare** | `cloudflare` | Cloudflare Tunnel (recommended for production) |

### Local Mode

Traefik binds to `127.0.0.1:38080`. Access tenants by adding hostname entries to `/etc/hosts`:

```
127.0.0.1  my-agent.agents.example.com
127.0.0.1  team-b.agents.example.com
```

### Cloudflare Tunnel Mode

The most secure production setup. Traefik stays local; Cloudflare handles TLS and DNS:

```bash
# Set in .env
INGRESS_MODE=cloudflare
CLOUDFLARE_TUNNEL_ID=<your-tunnel-id>
CLOUDFLARE_TUNNEL_CREDENTIALS_FILE=/path/to/credentials.json

# Render cloudflared config
./scripts/render-cloudflared-config.sh

# Install as a service (macOS)
./scripts/install-cloudflared-launchd.sh
```

See [Linux Cloud Host Quickstart](linux-cloud-host-quickstart.md) for the full Cloudflare setup.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Agent route returns 404 | Check `<AGENT>_ENABLED=1` in tenant env file |
| Health check fails | `docker compose logs` on the tenant container |
| Traefik not routing | Verify hostname in `deploy/tenants/tenants.yml` matches DNS |
| OpenClaw needs HTTPS | Use `localhost`, Cloudflare Tunnel, or set `OPENCLAW_ACCESS_MODE=trusted-proxy` |
| Filebrowser read-only | Enable write permissions in Filebrowser settings (OFF by default for security) |

For detailed diagnostics, use the host agent manager skill:

```bash
python3 skills/rundiffusion-host-agent-manager/scripts/validate_root_env.py
```
